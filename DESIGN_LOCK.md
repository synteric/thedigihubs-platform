# TheDigiHubs Approved UI Design Lock

These are the screens approved in the design conversation and used as the basis for the frontend code in this package.

## Approved screens

1. **Landing page**
   - Reference image: `clean_and_modern_procurement_platform_ui.png`
   - Notes: Ivalua-inspired section pacing, TheDigiHubs blue hub logo, 8-section landing page, CTA with woman image.

2. **Buyer dashboard**
   - Reference image: `modern_procurement_analytics_dashboard_interface.png`
   - Notes: collapsible sidebar without affecting logo, avatar dropdown with Sign out, buyer RFQ analytics and RFQ pipeline.

3. **Supplier dashboard**
   - Reference image: `supplier_dashboard_overview_thedigihubs.png`
   - Notes: visually distinct from buyer dashboard, uses Matched Opportunities, no global Submit Quote button, View Opportunities CTA.

4. **Admin dashboard**
   - Reference image: `admin_dashboard_overview_and_controls.png`
   - Notes: manage organizations, users, RFQs, membership plans, roles, tickets, revenue, audit logs, upgrades/downgrades.

5. **RFQ creation workflow**
   - Reference image: `saas_procurement_dashboard_rfq_workflow.png`
   - Notes: 7-step RFQ workflow, optional specifications/documents upload, platform supplier selection, external supplier invites by email.

## Important coding rules for Codex

- Do not redesign the brand or change the logo system.
- Do not add Alibaba references.
- Do not use “command center” or confusing infrastructure language.
- Keep the landing page header as: Logo, Solutions, Platform, Resources, Partners, Search, Language, Why TheDigiHubs, Contact Us.
- For Buyers and For Suppliers belong under Solutions.
- Supplier dashboard uses Matched Opportunities, not Open Opportunities.
- Submit Quote belongs on RFQ detail / quote preparation screens, not as a global supplier dashboard CTA.
- Sidebars should be collapsible without compressing or affecting the logo/header.
- Avatar/profile dropdown must include Sign out.
